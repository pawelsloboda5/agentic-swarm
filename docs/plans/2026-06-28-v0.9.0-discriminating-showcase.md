# Plan — v0.9.0: the discriminating showcase (a sharper held-out primary)

> **Status:** pre-flight DONE, pre-registration LOCKED (committed before arms). Remaining: build the scored
> arms → score → RESULTS → measurement-integrity ship-gate → PR (left for review). **Measured, not asserted:
> a null or a harness loss is reported verbatim.**

## Goal

Answer the v1.0-gating methodology question that v0.8.0 left open. v0.8.0 measured an honest **null**, but on a
**coarse** held-out primary ("does the game boot") that discriminated **zero** arms — so the null was an
absence of signal on a blunt ruler, not evidence the harness doesn't help. v0.9.0 builds a **sharper, pilot-
verified discriminating primary** and re-runs the clean comparison (architect harness vs a fair single-shot
control, identical spec) to see whether the harness's decomposition + gated integration + repair produces a
**measurably more correct** artifact — or, honestly, not.

## Why this can discriminate where v0.8.0 could not (the architecture)

- **Task:** a bespoke **deterministic integer simulation engine** ("Carousel", `SPEC.md`), not a game. The
  artifact is checked by **pure self-consistency** (a build vs *itself* via `hashState`/`snapshot`/`restore`)
  — no external oracle, no float/NaN marshaling, far less in-distribution than a spreadsheet or a game. This
  dissolves the integrity critic's three worst objections (oracle validity, marshaling, both-ace).
- **Primary (held-out, pilot-confirmed):** `F_FID` — **state fidelity under continuation** (restore a
  snapshot, run forward, must match running straight through). Strong single-shots consistently ship a
  **self-undetectable** save/load-completeness bug (snapshot omits PRNG/cooldown state): their own
  restore-*identity* self-test passes while restore-*continue* diverges. Pilot: single-shots 0.33, a correct
  reference 1.0 — real headroom, on a real bug.
- **Honest held-out framing:** the *requirement* is disclosed symmetrically (SPEC.md #2 says save/load must
  "behave identically going forward"); the **adversarial vectors + the rigor + the fact of measurement** are
  held out. SWE-bench-style hidden conformance. The hypothesis: a single shot *told* the requirement still
  fails it under one-pass attention, while the harness's dedicated persistence workstream + gated verification
  + repair may actually achieve it. If the harness's self-designed verification shares the same blind spot, it
  also fails → null. So the harness must genuinely verify better to win — not rigged.

## v1.0 implication (what this milestone feeds)

- **If the harness WINS** F_FID: the first measured evidence in the track that the harness's orchestration —
  not just rubric-in-prompt — adds correctness a fair single shot misses. v1.0 can make that *narrow,
  evidenced* claim ("the harness's gated verification measurably improves save/load fidelity, a dimension
  single shots reliably miss") — not a broad superiority claim.
- **If NULL/LOSS:** replicates the v0.8.0 finding on a *sharper, discriminating* instrument — strong evidence
  that on these tasks orchestration does not beat a fair single shot on artifact correctness. v1.0 then ships
  the harness for its **process** guarantees (parallel throughput, gated integration, bounded repair,
  auditability, forward-coupling) and explicitly does **not** claim higher artifact quality. Honesty over a
  manufactured win — the project's #1 value.

Either outcome is a publishable, honest result that moves the v1.0 decision forward on real data.
