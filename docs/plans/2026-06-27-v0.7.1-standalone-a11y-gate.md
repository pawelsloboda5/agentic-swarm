**Goal:** Ship the **full standalone `a11y` gate** the v0.7.0 design deferred — an auto-detected
axe-core / pa11y / Lighthouse runner sweep **scoped to a11y-distinctive rules** plus a separate-context
**static-semantics + keyboard critic**, backed by a zero-dep runner-output **normalizer** — completing
the v0.7 gate library without double-counting the cheap checks `ui-ux` already owns and without ever
rubber-stamping ("PASS ≠ conformance").

**Architecture:** A new markdown gate file `skills/architect/gates/a11y.md` (the 7-key schema, tier
`mixed`) + one new zero-dep Python module `skills/architect/gates/lib/a11y_report.py` that normalizes the
**heterogeneous, fail-closed** JSON each runner emits into one unified verdict. `a11y` GATES only on its
*distinctive* layer (the scoped runner sweep + the semantics/keyboard critic); the cheap static checks
(contrast, alt-text, accessible-name presence, tap-target) stay **single-owned by `ui-ux`** — `a11y`
partitions any overlapping-rule violations to **advisory**. When no runner is present (the default
zero-provision case) `a11y` **never borrows another gate's evidence to manufacture a pass**: the automated
sub-check is skipped-with-note, the browserless-capable semantics critic still gates, and if nothing
a11y-specific is verifiable the verdict is an honest `flag` — mirroring the `tests`/`ui-ux` precedent.

**Tech Stack:** One Python 3 stdlib module (CLI + importable funcs) tested by `pytest`; markdown gate +
reference + map edits loaded by the architect skill. Zero new plugin dependencies (axe/pa11y/Lighthouse +
a browser are **optional runtime-detected enhancers**). `claude plugin validate --strict` stays green both
modes. Version bump `0.7.0` → `0.7.1`.

## Design decisions reconciled from the planning critique (read before building)

These are the load-bearing corrections the 6-lens panel converged on. Every task below honors them.

1. **Single-owner decomposition (no double-count).** `ui-ux` remains the sole owner of WCAG contrast,
   alt-text, accessible-name *presence*, and tap-target. `a11y` owns ONLY: (a) a **scoped** automated
   runner sweep and (b) a **role/value/state + structure + keyboard** critic. `a11y` never *gates* on a
   ui-ux-owned check — overlapping-rule violations are surfaced **advisory** (deferred to ui-ux).
2. **Scoped runner gating set.** Running axe/pa11y with the WCAG tag set emits `color-contrast`,
   `image-alt`, `label`/`button-name`/`link-name`, `target-size` — all ui-ux-owned. The normalizer
   **excludes** those rule ids/codes from the *gating* count (partitions them to advisory). The excluded
   set is a documented module constant (`UIUX_OWNED_RULES`) — the explicit seam, also named in `a11y.md`.
3. **Browserless ⇒ flag, never a borrowed pass.** No runner present ⇒ the automated sub-check is
   SKIPPED-with-note (per the gate-runner contract); `a11y` does NOT re-run contrast as a floor. The
   browserless-capable static-semantics critic still gates; render-dependent parts (real tab-order
   reachability, visible-focus effectiveness) downgrade to **advisory**. If nothing a11y-specific can be
   verified ⇒ `status: flag` ("automated a11y not run; cannot objectively verify"), `degraded: true`,
   confidence ≤ 0.6 — never a `pass`. (Matches `tests.md`/`ui-ux.md`: absent prerequisite ⇒ flag.)
4. **Heterogeneous + fail-closed normalizer (the false-pass trap).** The runner JSON shapes differ by
   *invocation*, not just tool: pa11y's **CLI `--reporter json` emits a BARE ARRAY** (`[{type,...}]`)
   while its **JS API returns `{issues:[...]}`** (Context7 `/pa11y/pa11y` MIGRATION.md, verified
   2026-06-27); axe's programmatic `axe.run()` returns `{violations:[...]}` but `@axe-core/cli` can write
   a **top-level array of per-page result objects**. The normalizer accepts **both** the wrapper-dict and
   the bare/array forms and **fails closed**: unparseable / wrong-shape / missing-expected-keys ⇒ **exit 2
   + `error`**, NEVER `count==0 ⇒ pass`.
5. **Lighthouse is advisory + null-safe.** `categories.accessibility.score` is a *fractional weighted
   subset* → advisory only, **never a hard gate**. `score is None` OR a top-level `runtimeError` ⇒
   degraded/error (surface it), never coerced to 0, never advisory-pass.
6. **Exit-code convention mirrors `wcag_contrast.py`:** `0` pass · `1` fail (gating violations) · `2`
   bad/unreadable input. Decide pass/flag on raw counts; ASCII-only stdout (cp1252-safe).
7. **Tier-label drift surfaces (the guard).** `tests/test_gate_library.py::test_tier_labels_consistent_across_docs`
   checks 4 surfaces against a hardcoded gate tuple. Flipping `a11y` to active means updating **all** of:
   the gate-map Status+Notes (Notes cell must *literally begin* with `mixed`), `gate-runner.md`'s starter
   table **and** its "deferred to v0.7.x" note, `SKILL.md`'s starter-library annotation, the CHANGELOG,
   **and** adding `a11y` to the guarded gate tuple — in lockstep, or the suite goes red.
8. **PATCH-not-fix framing.** `0.7.1` is correct (pre-1.0; 0.7.0 publicly reserved a11y for "v0.7.x";
   0.8.0 already committed to the showcase), but the CHANGELOG must say it's an **additive feature shipped
   as a patch to complete v0.7.0's originally-scoped a11y**, so PATCH doesn't read as a bugfix.

**Out of scope → v0.8.0:** the measured Three.js showcase (see the companion plan). v0.7.1 stays
browserless-CI-safe and zero-provision; the live runner only fires under v0.8's provisioned browser.

**Step 1 — Write the failing tests** (`tests/test_a11y_report.py`). Cover heterogeneity, the
ui-ux-owned partition, Lighthouse null-safety, and fail-closed. Vectors mirror the Context7-verified shapes.

```python
import json
import pytest
import a11y_report as a

# --- axe: programmatic object form {violations:[{id,nodes}]} ---
def test_axe_object_form_counts_gating_violations():
    data = {"violations": [
        {"id": "landmark-one-main", "nodes": [{}]},          # a11y-distinctive -> gates
        {"id": "color-contrast", "nodes": [{}, {}]},          # ui-ux-owned -> advisory
    ], "incomplete": [{"id": "color-contrast", "nodes": [{}]}]}
    v = a.normalize(data, "axe")
    assert v["runner"] == "axe"
    assert v["gating_violations"] == 1          # only landmark-one-main gates
    assert v["advisory_violations"] == 1        # color-contrast partitioned out
    assert v["incomplete_count"] == 1           # surfaced, not gating
    assert v["pass"] is False                   # 1 gating violation
    assert v["error"] is None

# --- axe: CLI array-of-pages form [{violations:[...]}, ...] ---
def test_axe_cli_array_form_sums_pages_and_passes_clean():
    data = [{"violations": []}, {"violations": []}]
    v = a.normalize(data, "axe")
    assert v["gating_violations"] == 0 and v["pass"] is True

# --- pa11y: JS API object form {issues:[...]} ---
def test_pa11y_object_form_errors_only():
    data = {"issues": [
        {"code": "WCAG2AA.Principle1.Guideline1_3.1_3_1.H42.2", "type": "error"},   # structure -> gates
        {"code": "WCAG2AA.Principle1.Guideline1_4.1_4_3.G18", "type": "error"},      # 1_4_3 contrast -> advisory
        {"code": "X", "type": "warning"},                                            # warning -> advisory
    ]}
    v = a.normalize(data, "pa11y")
    assert v["gating_violations"] == 1          # only the structure error gates
    assert v["pass"] is False

# --- pa11y: CLI BARE ARRAY form [{type,...}] (the false-pass trap) ---
def test_pa11y_cli_bare_array_is_not_a_false_pass():
    data = [{"code": "WCAG2AA.Principle1.Guideline1_3.1_3_1.H42.2", "type": "error"}]
    v = a.normalize(data, "pa11y")
    assert v["gating_violations"] == 1 and v["pass"] is False    # NOT silently 0/pass

# --- ui-ux-owned-only input gates clean (deferred to ui-ux) ---
def test_only_uiux_owned_violations_do_not_gate_a11y():
    data = {"violations": [{"id": "color-contrast", "nodes": [{}]},
                           {"id": "image-alt", "nodes": [{}]},
                           {"id": "target-size", "nodes": [{}]}]}
    v = a.normalize(data, "axe")
    assert v["gating_violations"] == 0 and v["advisory_violations"] == 3 and v["pass"] is True

# --- lighthouse: advisory score, never a hard gate ---
def test_lighthouse_valid_score_is_advisory_not_gating():
    data = {"categories": {"accessibility": {"score": 0.82}}}
    v = a.normalize(data, "lighthouse")
    assert v["score"] == 0.82 and v["advisory"] is True
    assert v["pass"] is True                    # advisory score never hard-fails
    assert v["degraded"] is True                # lighthouse-only => no hard gate => degraded

def test_lighthouse_null_score_is_degraded_not_zero():
    v = a.normalize({"categories": {"accessibility": {"score": None}}}, "lighthouse")
    assert v["score"] is None and v["degraded"] is True and v["error"] is not None

def test_lighthouse_runtime_error_is_degraded():
    v = a.normalize({"runtimeError": {"code": "NO_FCP"},
                     "categories": {"accessibility": {"score": 0.0}}}, "lighthouse")
    assert v["degraded"] is True and v["error"] is not None

# --- fail-closed on garbage / wrong-shape ---
def test_garbage_json_does_not_pass():
    for bad in [42, "nope", {"unexpected": 1}, {"violations": "notalist"}]:
        v = a.normalize(bad, "axe")
        assert v["error"] is not None and v["pass"] is False    # never count==0 => pass

def test_unknown_runner_errors():
    with pytest.raises(ValueError):
        a.normalize({"violations": []}, "bogus")

# --- CLI exit codes mirror wcag_contrast.py (0 pass / 1 fail / 2 bad-input) ---
def test_cli_exit_codes(tmp_path):
    p = tmp_path / "axe.json"
    p.write_text(json.dumps({"violations": []}), encoding="utf-8")
    assert a._main([str(p), "--runner", "axe"]) == 0
    p.write_text(json.dumps({"violations": [{"id": "tabindex", "nodes": [{}]}]}), encoding="utf-8")
    assert a._main([str(p), "--runner", "axe"]) == 1
    p.write_text("{ not json", encoding="utf-8")
    assert a._main([str(p), "--runner", "axe"]) == 2
```

**Step 2 — Run, see it fail:** `python -m pytest tests/test_a11y_report.py -q` → FAIL (module missing).

**Step 3 — Implement** `a11y_report.py` (Python stdlib only). Public surface:
- `UIUX_OWNED_RULES` — frozenset of axe rule ids ui-ux owns: `color-contrast`, the image-alt family
  (`image-alt`, `input-image-alt`, `area-alt`, `object-alt`, `svg-img-alt`, `role-img-alt`), the
  accessible-name family (`label`, `label-title-only`, `button-name`, `link-name`,
  `aria-input-field-name`, `aria-toggle-field-name`, `aria-command-name`), and `target-size`. Plus
  `UIUX_OWNED_SC` — WCAG SC substrings for pa11y codes: `1_4_3` (contrast), `1_1_1` (non-text),
  `4_1_2` (name-role-value *name* part), `2_5_8` (target size). **Document both as the ui-ux seam.**
- `normalize(data, runner) -> dict` returning the unified verdict
  `{runner, gating_violations, advisory_violations, errors_only, score, incomplete_count, pass,
  degraded, advisory, error, note}`. Per-runner:
  - **axe:** accept a dict (`data.get("violations")`) OR a list of page dicts (sum each page's
    `violations`); a violation gates iff `id not in UIUX_OWNED_RULES`; sum `incomplete` length into
    `incomplete_count` (surfaced, non-gating). `pass = (error is None and gating_violations == 0)`.
  - **pa11y:** accept a dict (`data.get("issues")`) OR a bare list; keep only `type == "error"`
    (`errors_only`); an error gates iff its `code` contains no substring in `UIUX_OWNED_SC`.
  - **lighthouse:** read `categories.accessibility.score`; if `runtimeError` present or `score is None`
    ⇒ `error="lighthouse run errored or score missing"`, `degraded=True`, `pass=False`; else
    `advisory=True`, `degraded=True` (LH alone never hard-gates), `pass=True` (advisory never fails).
  - **fail-closed:** wrong root type, `violations`/`issues` not a list, or missing expected keys ⇒
    `error="unrecognized/malformed <runner> JSON"`, `degraded=True`, `pass=False`. Unknown `runner` ⇒
    `raise ValueError`.
- `_main(argv) -> int`: `python a11y_report.py <json-file> --runner axe|pa11y|lighthouse`; reads + parses
  (JSONDecodeError ⇒ return 2), prints the verdict as **ASCII-only** JSON, returns `0` (pass) / `1`
  (gating fail) / `2` (bad/unreadable input). `if __name__ == "__main__": sys.exit(_main(sys.argv[1:]))`.

**Step 4 — Run, see it pass:** `python -m pytest tests/test_a11y_report.py -q` → PASS.

**Step 5 — Commit:** `feat(gates): zero-dep a11y runner-output normalizer (TDD)`.

## Task 2: `a11y.md` — the standalone gate file (7-key)

**Files:** Create `skills/architect/gates/a11y.md` — mirror the heading layout of
`gates/ui-ux.md` (`## Definition` with the 7 bolded keys, then `## Evidence required for a pass`,
then `## On fail`). The structural tests require literal `**id:**` … `**backing_skill:**` markers and a
tier in `{objective, critic, advisory, mixed}`.

Content (the 7 keys):
- **id:** `` `a11y` `` (must match filename for `test_each_gate_id_matches_filename`).
- **applies_when:** the workstream renders HTML / web UI **and a deeper-than-cheap a11y bar is wanted** —
  distinct from `ui-ux`'s folded cheap a11y. (Cross-reference: contrast / alt-text / accessible-name
  presence / tap-target are **owned by `ui-ux`**; this gate does **not** re-gate them.)
- **tier:** `` `mixed` `` (objective scoped-runner floor + a separate-context semantics/keyboard critic).
- **criteria:**
  - **Objective floor — scoped automated runner sweep (browser-dependent; OPTIONAL enhancer):**
    auto-detect a runner (priority `axe` → `pa11y` → `lighthouse`), run it against the rendered output,
    feed its JSON to `gates/lib/a11y_report.py`. **Gate iff `gating_violations == 0`** (axe/pa11y).
    Violations of **ui-ux-owned rules** (`UIUX_OWNED_RULES` / `UIUX_OWNED_SC` — list them here as the
    seam) are **advisory only** (deferred to ui-ux). **Lighthouse score is advisory, never a hard gate.**
    **No runner present ⇒ SKIP this sub-check with a visible note** ("runner unavailable; automated a11y
    not run") — never a fail-because-absent, **never a borrowed-contrast pass.**
  - **Critic rung — static-semantics + keyboard (separate context; the `verifier`):** landmark + heading
    structure, programmatic form-label association, **role / value / state ARIA** correctness
    (explicitly **excluding** accessible-name *presence*, which ui-ux owns), reading/DOM order. The
    **statically checkable** parts run **browserless and GATE**; the **render-dependent** parts (actual
    tab-order reachability, visible-focus effectiveness) **downgrade to advisory** when no browser
    (mirroring ui-ux's screenshot-critic downgrade — state this split explicitly so it doesn't read as
    re-gating ui-ux's `:focus-visible`-defined static check).
- **verifier:** the separate-context semantics/keyboard critic prompt above — binary per-criterion, one
  grounded pass, citations; the producer never grades itself.
- **confidence:** **never `1.0` on automation alone** (PASS ≠ conformance — axe/pa11y cover ~30–50% of
  WCAG SC). High when a runner ran + the critic gated; **≤ 0.6 + `degraded: true`** when no runner and
  only the static-semantics critic ran; an honest **`flag`** when nothing a11y-specific is verifiable.
- **backing_skill:** `responsive-ui-audit` (Playwright + axe), `wcag-aa-contrast-remediation`,
  `web-design-guidelines` — **optional enhancers only**; the bundled path runs with zero external skills.

`## Evidence required for a pass`: the runner command + `a11y_report.py` verdict (gating_violations==0,
the partitioned advisory count, incomplete_count) **and** the critic's per-criterion booleans with
citations. **`## PASS ≠ conformance`** subsection: an automated green + passing critic is **necessary,
not sufficient**; full conformance needs a manual audit; confidence never reaches 1.0 on automation alone.
`## On fail`: re-brief with the exact unmet criteria (bounded N=2 per the runner contract), then honest
`flag` enumerating `unmet_criteria`; advisory items never re-run.

**Verify validate** (both modes). **Commit:** `feat(gates): standalone a11y gate (scoped runner + semantics critic)`.

## Verification gate (before any merge/tag)

Re-run the tiered ship-gate that v0.5/v0.6/v0.7 used:
- **Objective floor:** `pytest` + `node --test` + `validate --strict` (both modes); PR mergeable + CI green.
- **Critic tier** (a `Workflow` of fresh-context adversarial lenses, retry wave, armed watchdog):
  **design-fidelity, portability-antitheater, honesty, mechanics-integrity, + `a11y-correctness`** (does
  the normalizer actually fail-closed and partition ui-ux-owned rules to advisory? does the browserless
  path flag rather than borrow-pass?). Loop until **zero blockers**, fixing each real finding and adding a
  guard test where a class could recur.
