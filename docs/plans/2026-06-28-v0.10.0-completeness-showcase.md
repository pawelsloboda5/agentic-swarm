# Plan — v0.10.0: the completeness-under-scale showcase

> **Status:** Phase-0 design CONVERGED (design swarm + adversarial critic REVISE→resolved); instrument
> BUILT + VALIDATED; pre-flight (ambiguity-scrub + discrimination check) IN PROGRESS. Remaining:
> lock prereg → build scored arms → score → RESULTS → ship-gate → PR (left for review).
> **Measured, not asserted: a null/loss is reported verbatim. The most probable outcome is a third
> honest null, and the design is built to report it under the correct mechanism label.**

## Goal

Test the **one dimension** the two prior showcases (v0.8.0 game, v0.9.0 engine) never exercised — both
measured honest **nulls** on *correctness of one moderate artifact*, which strong single shots handle
very well. This milestone tests the harness's **strongest theoretical case: completeness under scale** —
a single artifact with **40 mutually-independent requirements** where a single shot's finite attention
might make it **silently drop** some, and the harness's disjoint-slice decomposition (one focused worker
per requirement group) + gated integration could directly close the gap.

## What makes this honest (the critic's three resolved blockers)

A Phase-0 adversarial measurement-integrity critic returned **REVISE** with three blockers; all are
resolved in the design + instrument (see `evals/loop-demo/lib/CRITIQUE.md`):

1. **Per-export scorer isolation** — each export is scored in its own child `node` process with an 8 s
   per-export timeout, so a non-terminating export costs **exactly 1/40** (verified: `selftest-hang`
   scores 0.975, the hang isolated). Without this, one buggy export would zero the whole arm and destroy
   the per-requirement independence the design rests on.
2. **Empirical ambiguity-scrubber** — an independent SPEC-only reader predicts every held-out vector
   blind; divergences are dropped/re-pinned (the `correct-ref ≥ 0.99` gate is author-self-consistent and
   cannot catch author-side ambiguity). Plus the 5 highest-ambiguity requirements were **dropped**.
3. **Decomposed metric** — `OMISSION-completeness` / `EDGE-correctness` / `CASCADE-isolation` are scored
   and reported separately, and **only OMISSION-completeness carries the "completeness under scale"
   claim**. A checklist-fed single shot rarely omits enumerated exports, so any gap is most likely on
   edges (per-slice focus) or shared-helper cascades — *different* mechanisms, reported under their own
   labels, never mislabeled as drop-prevention.

## Architecture (the instrument)

- **Artifact:** one zero-dep, offline ES module `lib.mjs`, **40 named pure exports**, string/integer I/O
  only (no float/NaN marshaling). A "deterministic wire-format & canonical-form toolkit": reversible
  codecs (round-trip), involutions, idempotent canonicalizers, and pinned formatters — across **6
  disjoint workstreams** (the decomposition the harness will use). Each requirement pinned to **exactly
  one output** (RFC 4648/3986/4180/5952/4122 cited; custom base32 alphabet + author-pinned rules fully
  disclosed in `SPEC.md` with 2 happy examples each).
- **Held-out scorer (pure node, zero provisioning):** two axes per export — **A1 property** (round-trip /
  idempotence / involution over a seeded hidden corpus; reference-free) and **A2 spec-vectors** (186
  held-out edge `input→exact-output` pairs; the pinned-but-unexemplified edges). Composed into the three
  decomposed metrics by `aggregate.py`; `verdict.py` applies the rule mechanically.
- **Two reference anchors (validated):** correct reference = **1.000** all axes; incomplete reference (8
  stubs + 6 edge-botches) = **OMISSION 0.800 / EDGE 0.908 / COMPLETE 0.650** → the ruler measures
  coverage, not schema. (`scoring/instrument-validation.json`.)

## The decision rule (decomposed; instrument-authoritative)

- **HEADLINE (OMISSION axis):** HARNESS WINS iff the axis is **live** (control omitted ≥1 export) AND
  `(omission_h − omission_c) ≥ MARGIN` AND the gap **exceeds the within-arm spread**. Else **NULL** on
  the headline (the expected outcome). Degenerate **ceiling** (both ≥ 0.975) → NULL ("single shots did
  not drop; the mechanism had nothing to act on").
- **SECONDARY:** report `EDGE-correctness` delta ("per-slice focus aids edge correctness") and
  `CASCADE-isolation` ("decomposition avoids shared-helper cascades") — separate findings, never the
  headline. **LOSS** iff control composite beats harness by ≥ MARGIN.
- **MARGIN** target **+0.10** (≈ 4/40), **pilot-calibrated then frozen** in `PREREGISTRATION.md`; report
  within-arm spread + McNemar discordant pairs (K=3 honesty).

## Invariants preserved

Plugin stays **zero-dependency** (the scorer is pure node — no Playwright, no `node_modules`; even cleaner
than v0.9.0). Skills frontmatter unchanged. `validate --strict` green both modes. No secrets, no
`.claude/`. Pre-registration **never amended** after the freeze commit (a necessary instrument fix is a
disclosed deviation with an updated hash). **Never merge/tag/release without the user.**

## v1.0 implication (what this milestone feeds)

- **If the harness WINS the OMISSION axis** (unlikely per the critic): first measured evidence that
  decomposition prevents silent requirement drops at scale — a *narrow, evidenced* v1.0 claim.
- **If a SECONDARY axis shows signal** (edge-correctness or cascade-isolation): a real but *different*
  benefit, reported under its correct label — informs a *scoped* v1.0 claim ("decomposition avoids
  shared-helper cascades / aids edge correctness"), not "completeness under scale."
- **If NULL across all axes** (most probable): a **third** honest null on a *third* task family with a
  *validated, decomposed* instrument → strong evidence to **narrow** v1.0 to the harness's **process**
  guarantees (parallel throughput, gated integration, bounded repair, auditability) and explicitly **not**
  claim artifact-quality/completeness uplift. Honesty over a manufactured win — the project's #1 value.

Either outcome is a publishable, honest result that moves the v1.0 decision forward on real data.
