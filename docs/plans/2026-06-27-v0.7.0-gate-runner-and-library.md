**Goal:** Ship the **gate runner** (the §2 contract that turns a gate definition into a verdict) and the
**starter gate library `{tests, assets, ui-ux}`** (each a self-contained 7-key gate file with cheap a11y
folded into `ui-ux`), plus the shared zero-dep WCAG-contrast util they rely on — so the architect's
Phase 3 can actually *run* the gates it forward-couples into briefs in Phase 1.

**Architecture:** Gates are **markdown definition files** (`skills/architect/gates/<id>.md`) declaring the
7 keys `{id, applies_when, tier, criteria, verifier, confidence, backing_skill}`. The **gate runner** is
a **markdown contract** (`skills/architect/reference/gate-runner.md`) the orchestrator follows — LOAD →
DETECT backing skill (zero-dep fs probe) → RUN BY TIER → EMIT one verdict per applicable gate → enforce
the anti-theater invariant → on fail re-brief bounded N=2 → degrade gracefully (never silently). The only
genuinely mechanical, reusable bit ships as **real zero-dep code**: a WCAG-contrast util
(`skills/architect/gates/lib/wcag_contrast.py`, Python stdlib) that both the `ui-ux` gate's objective
floor invokes and the test suite unit-tests with exact vectors.

**Tech stack:** Markdown gate/runner files (frontmatter-free; loaded by the architect skill) + one Python
stdlib module (the WCAG util, CLI + importable funcs) tested by `pytest` (the repo's existing infra).
Zero new dependencies. `claude plugin validate --strict` must stay green both modes.

**Scope (tight):** v0.7.0 = runner + `{tests, assets, ui-ux}` + WCAG util + Phase-3 wiring + tests +
release. **Out of scope → v0.7.x:** the full standalone `a11y` gate (axe/pa11y/Lighthouse runner +
keyboard/semantics critic — re-verify those APIs via Context7 at that build time). v0.8 = measured showcase.

## Task 1: Shared WCAG-contrast util (TDD — exact math)

**Files:**
- Create: `skills/architect/gates/lib/wcag_contrast.py`
- Test: `tests/test_wcag_contrast.py`
- Modify: `tests/conftest.py` (add `skills/architect/gates/lib` to `sys.path`)

The formula is the fixed W3C standard (no library; from mvp-plan §4):
`cs=c/255; lin = cs/12.92 if cs<=0.03928 else ((cs+0.055)/1.055)**2.4; L=0.2126R+0.7152G+0.0722B;
ratio=(L_light+0.05)/(L_dark+0.05)`. Thresholds: normal ≥4.5, large (≥24px or ≥18.66px bold) ≥3.0,
UI/graphics ≥3.0.

**Step 1 — failing test** (`tests/test_wcag_contrast.py`): exact anchors + symmetry + thresholds:
```python
import wcag_contrast as w
def test_black_on_white_is_21():     assert round(w.ratio("#000000", "#ffffff"), 2) == 21.0
def test_white_on_white_is_1():      assert round(w.ratio("#ffffff", "#ffffff"), 2) == 1.0
def test_symmetric():                assert w.ratio("#123456", "#abcdef") == w.ratio("#abcdef", "#123456")
def test_passes_aa_normal_boundary():
    # #767676 on white is the canonical darkest grey that passes AA normal text (~4.54)
    r = w.ratio("#767676", "#ffffff"); assert 4.5 <= r <= 4.6 and w.passes(r, "normal")
def test_fails_aa_normal_below_threshold():
    assert not w.passes(4.49, "normal") and w.passes(4.5, "normal")
def test_large_text_threshold_is_3():
    assert w.passes(3.0, "large") and not w.passes(2.99, "large")
def test_rejects_bad_hex():
    import pytest
    with pytest.raises(ValueError): w.ratio("#xyz", "#ffffff")
```
**Step 2 — run, see it fail:** `python -m pytest tests/test_wcag_contrast.py -q` → FAIL (module missing).
**Step 3 — implement** `wcag_contrast.py`: `parse_hex(s)->(r,g,b)` (accept `#rgb`/`#rrggbb`, raise
ValueError otherwise), `_lin(c)`, `luminance(rgb)`, `ratio(fg,bg)->float`, `passes(ratio, size)` with
size ∈ {normal,large,ui} → thresholds {4.5,3.0,3.0}, and a `__main__` CLI:
`python wcag_contrast.py <fg> <bg> [normal|large|ui]` → prints JSON `{ratio, pass, threshold}` and exits
0 if pass else 1 (so the gate can use it as an objective machine check). ASCII-only output (cp1252-safe).
**Step 4 — run, see it pass.** **Step 5 — commit:** `feat(gates): zero-dep WCAG contrast util (TDD)`.

## Task 2: Gate-runner contract reference

**Files:**
- Create: `skills/architect/reference/gate-runner.md`

**Step 1 — write the contract** (markdown the orchestrator follows), implementing mvp-plan §2 verbatim:
- The `gate` schema (7 keys) and the `verdict` schema
  (`{gate_id, tier, status∈{pass,flag,fail}, confidence 0..1, backing_skill, skill_used, degraded,
  evidence[], unmet_criteria[]}`).
- **LOAD + validate** the gate file (all 7 keys; malformed ⇒ skip with a logged warning, not silent drop).
- **DETECT `backing_skill`** — zero-dep fs probe on the on-disk **directory** name at
  `~/.claude/skills/<id>/SKILL.md`, `.claude/skills/<id>/SKILL.md`,
  `~/.claude/plugins/cache/**/skills/<id>/SKILL.md`; optional, non-load-bearing cross-check of
  `claude plugin list` / `enabledPlugins` (never parse-depend).
- **RUN BY TIER:** `objective` = run the bundled machine command, status off exit/threshold; `critic` =
  spawn a **separate-context** judge (binary per-criterion + one grounded pass + citations; producer
  never grades itself; no re-score loop); `advisory` = surface only. A present `backing_skill` is an
  enhancer **over** (never instead of) the bundled path.
- **EMIT** exactly one verdict per applicable gate.
- **INVARIANT (anti-theater):** no `status: pass` without ≥1 `evidence` + a `tier` + a `confidence`.
- **ON FAIL:** re-brief the workstream with the **exact** `unmet_criteria` and re-run, **bounded N=2**;
  then honest `flag`; confidence reflects attempts-exhausted. Advisory fails never re-run.
- **GRACEFUL ≠ SILENT:** missing `backing_skill` ⇒ run bundled criteria, `degraded:true` + lowered
  confidence + visible note; missing optional runner binary ⇒ **skip** that sub-check (not fail) + note.

**Step 2 — verify validate (both modes). Step 3 — commit:** `feat(gates): gate-runner contract reference`.

## Task 3: `tests` gate (objective)

**Files:** Create `skills/architect/gates/tests.md`.

7-key gate file. `id: tests`; `applies_when`: output includes runnable code; `tier: objective`;
`criteria`: detect runner from manifests (package.json/pyproject/cargo/go.mod) → run the project's own
test command → **exit 0 AND >0 tests collected** (zero tests = FAIL, the #1 test theater) + typecheck/build
clean + **diff-coverage presence** (every changed source file has an associated test); coverage % only if
a baseline exists. `verifier`: a separate-context check that the suite actually exercises the changed code,
not just that it's green. `confidence`: high when the runner is detected + tests collected. `backing_skill`:
`gsd-add-tests` (auto-gen missing tests) / `superpowers:test-driven-development` (enhancers only).
**Verify validate. Commit:** `feat(gates): tests gate (objective)`.

## Task 5: `ui-ux` gate (mixed; uses the WCAG util)

**Files:** Create `skills/architect/gates/ui-ux.md`.

`id: ui-ux`; `applies_when`: output renders UI; `tier`: mixed. **Objective floor (browser-free, always

with reported confidence. One grounded pass, binary per-criterion, citations. `backing_skill`:
