## Measurement-honesty corrections folded in (read before building)

The planning panel rejected the first showcase design as **biased by construction**. The fixes:

1. **Teaching-to-the-test is real and must be neutralized.** The harness arm's briefs *forward-couple the
   exact gate criteria* AND the gate-runner does **N=2 gate-driven repair** against them; the committed
   baselines saw neither. Scoring all arms with that same ruler measures "we told one builder the answer
   key," not methodology. **Fix — do both:**
   - **(a) Fair second arm:** run a **fresh bare-`/loop` arm under a prompt that carries the IDENTICAL
     acceptance criteria in-prompt** (so both arms *know* the rubric; the only remaining difference is the
     methodology — decomposition + gated repair vs single-shot). This isolates the variable the showcase
     actually claims.
   - **(b) Held-out primary metric:** the **pre-registered primary metric is a check that appeared in NO
     brief** (e.g. a held-out a11y/keyboard probe, or a playability assertion). In-brief gate pass-rates
     are reported as **secondary / "conformance to spec we asked for,"** never as the headline uplift.
   - Disclose the **retry-asymmetry** (harness got ≤3 gate-driven attempts per gate; baselines got one
     shot, no feedback) in every table caption.
2. **Pre-register before building.** Commit `evals/loop-demo/game/RESULTS.md` **scaffold first** with
   exactly **one primary metric**, its **uplift threshold**, and a **numeric definition of a null/weak
   result** — *before* any arm runs. All other readouts are labeled secondary/exploratory. (Closes the
   multiple-comparisons / pick-the-winner-after-the-fact hole at n=1.)
3. **Reproducibility ≠ rigor anchor.** Re-scoring 3 fixed files deterministically varies *nothing* — it
   is **not** the v0.5 curve (which varied drop-rate × seeds). State plainly: *"objective-floor scoring is
   deterministic / re-runnable bit-identically; the COMPARISON is n=1, uncontrolled, one build per arm —
   no statistical inference, no generalization."* Scope the determinism claim to the **objective/automated
   metrics only** (critic rungs are non-deterministic).
4. **Honest baseline labels + named confounds.** `evals/loop-demo/agentic-swarm/index.html` (1476 lines)
   was the v0.5 **naive-swarm** arm, **not** a bare `/loop` — call the two committed games the **"two
   committed pre-harness baselines."** The harness-vs-baseline delta changes **≥4 variables** (plugin
   on/off, `/loop` vs architect harness, criteria-in-brief vs not, Playwright-provisioned vs not) — name
   them; do not imply a clean single-variable uplift. The **fair second arm (1a)** is what restores a
   single-variable comparison for the headline claim.
5. **Provision the runner, not just the browser.** Playwright is a driver; the **a11y runner**
   (axe-core/pa11y/Lighthouse) is what emits the JSON `a11y_report.py` parses. Provision **both**.
6. **Playability is an auxiliary metric, not a library gate.** There is no `playability` gate — score it
   as a **separate-context BINARY critic with citations** (not a holistic score), labeled auxiliary, kept
   out of the "same gate-runner" instrument framing.
7. **Confirm metrics discriminate.** The shared prompt mandates **procedural geometry only**, so the
   `assets` gate (placeholder-absence + favicon) may pass trivially for all arms — **verify it
   discriminates before reporting it**, else drop it as degenerate.
