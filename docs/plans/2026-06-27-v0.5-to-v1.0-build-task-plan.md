## v0.7.0 — Starter gate library + gate-runner  🟡 (the crux)

**Goal:** the MVP gate set `{ tests, assets, ui-ux }` (each with folded objective a11y sub-checks) +
the full standalone `a11y` gate + a gate-runner implementing the §2 contract. Every gate ships
**self-contained**, declares a `tier`, and **reports `confidence`** (the anti-theater invariant). Each
gate file carries all 7 keys: `id / applies_when / tier / criteria / verifier / confidence / backing_skill`.

| Task | Files | Sub-steps |
|---|---|---|
| **0.7-T1** Shared WCAG contrast util | `skills/architect/gates/lib/wcag-contrast.*` | Zero-dep inline implementation (sRGB→linear→relative luminance→ratio; pass ≥4.5 normal / ≥3 large or UI). Shared by `ui-ux` + `a11y`. **Re-verify the formula via Context7 at build time** (CLAUDE.md discipline). |
| **0.7-T2** `tests` gate (objective) | `skills/architect/gates/tests.md` | Detect runner from manifests → run project's own test cmd → **exit 0 AND >0 tests collected** (zero-tests = FAIL) + typecheck/build + diff-coverage presence. Enhancers: `gsd-add-tests`, `superpowers:test-driven-development`. |
| **0.7-T3** `assets` gate (mixed) | `skills/architect/gates/assets.md` | ripgrep placeholder/lorem/empty-src sweep (scoped + allowlist) + stdlib stat (local assets exist & >0 bytes) + SVG well-formed XML + favicon for HTML. Remote refs + "AI-filler" = advisory. Enhancers: `responsive-ui-audit`, `frontend-design`. |
| **0.7-T4** `ui-ux` gate (mixed) | `skills/architect/gates/ui-ux.md` | **Objective floor (browser-free, always gates):** WCAG contrast, spacing scale, hover+focus-visible, breakpoints, anti-AI-default heuristic + folded a11y (alt-text, accessible-name, tap-target ≥24px). **Critic rung:** separate-context screenshot critic (3 breakpoints) at high confidence *if a browser exists*, else static critic downgraded to **advisory** with reported confidence. One grounded pass, binary per-criterion, citations. |
| **0.7-T5** `a11y` gate (full, mixed) | `skills/architect/gates/a11y.md` | Bundled WCAG math + alt/label/tap-target + auto-detected axe-core/pa11y/Lighthouse (`violations===0`) + keyboard/focus/semantics critic. **"PASS ≠ conformance"** caveat. **Re-verify axe/pa11y/Lighthouse APIs via Context7 at build time** (only axe-core is Context7-confirmed; the rest are official-docs-only). |
| **0.7-T6** Gate-runner | `skills/architect/reference/gate-runner.md` (+ any zero-dep helper) | Implement the §2 contract: LOAD+validate (7 keys; malformed ⇒ skip+warn) → **zero-dep skill-detection probe** matching the **on-disk skill *directory* name** (not the `/plugin:skill` command string) at `~/.claude/skills/<id>/SKILL.md`, `.claude/skills/<id>/SKILL.md`, `~/.claude/plugins/cache/**/skills/<id>/SKILL.md`; **optional, non-load-bearing** cross-check of `claude plugin list` / `.claude/settings.json` `enabledPlugins` (**never depend on parsing them**) → RUN BY TIER (objective=machine cmd; critic=separate-context judge, binary+citations, no re-score loop; advisory=surface) → EMIT one verdict per applicable gate → **anti-theater invariant: no `pass` without ≥1 evidence + tier + confidence** → ON FAIL re-brief with exact unmet criteria, bounded N=2, then honest `flag` → **graceful ≠ silent**, two cases: (i) missing `backing_skill` ⇒ run bundled criteria, `degraded:true` + lowered confidence + note; (ii) **missing optional runner binary** (axe/pa11y/Lighthouse/Playwright) ⇒ **skip that sub-check, never fail**, with an explicit "runner unavailable; bundled checks only" note. |
| **0.7-T7** Phase-3 wiring | `skills/architect/SKILL.md` | Wire gated integration (Phase 3) to invoke the gate-runner per workstream against its selected gates; iterate failures until pass or explicit flag. |
| **0.7-T8** Tests + validate + release | `tests/`, manifests | Invariant tests for the verdict schema + anti-theater rule + detection probe. `validate --strict` green. Bump → `0.7.0` + CHANGELOG + tag (⏏). |
