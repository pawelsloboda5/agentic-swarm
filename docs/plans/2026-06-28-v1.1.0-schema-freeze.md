# v1.1.0 — Freeze the public contract (gate + verdict schema) + SemVer policy

> **Status:** scope **confirmed by the owner** (2026-06-28, plan-then-confirm; version = **1.1.0**) ·
> endpoint = **PR left for review** (then ship on explicit go-ahead). This finishes the originally-planned
> v1.0 `1.0-T2` (the gate-schema freeze) that was deferred out of `1.0.0`.

## Goal

Make the plugin's public surface a **stable, declared, CI-enforced contract** so it can carry a real SemVer
promise post-1.0. The schema was already *defined* (`gate-runner.md`) and *partly* guarded
(`test_gate_library.py`); this milestone (1) **declares it frozen**, (2) **documents the SemVer policy**, and
(3) **locks the exact surface against drift** with a guard test, so changing it forces a deliberate version
decision.

## The frozen public contract

1. **Skill commands** — `/agentic-swarm`, `/agentic-swarm:architect`, `/agentic-swarm:as-new-project`.
2. **Gate-file schema** — 7 keys (`id`, `applies_when`, `tier`, `criteria`, `verifier`, `confidence`,
   `backing_skill`); `tier` ∈ {`objective`, `critic`, `advisory`, `mixed`}.
3. **Verdict schema** — `gate_id`, `tier`, `status`, `confidence`, `backing_skill`, `skill_used`,
   `degraded`, `evidence[]`, `unmet_criteria[]`; `status` ∈ {`pass`, `flag`, `fail`}; `confidence` ∈ 0.0..1.0.

Canonical definition: `skills/architect/reference/gate-runner.md` § Schemas. Implementation helpers under
`skills/architect/gates/lib/` (the WCAG / a11y CLIs) are **not** part of the frozen surface.

## Post-1.0 SemVer policy

| Change | Bump |
|---|---|
| Remove/rename a gate-schema or verdict key; remove a `tier`/`status` value; remove/rename a skill or command | **MAJOR** |
| Add a gate / skill / **optional** schema key (safe default) / enhancer | **MINOR** |
| Fix, doc clarification, criteria wording, test/CI change | **PATCH** |

## Why 1.1.0 (not 1.0.1)

The freeze adds a **new public guarantee** (the stability contract + the SemVer policy) and a **new CI
guard** — a backward-compatible *additive* change with no schema/behavior change ⇒ SemVer **MINOR**. (A
PATCH was defensible as "docs+tests only"; the owner chose MINOR for the clearer "we now promise stability"
signal.)

## Invariants (CI-enforced)

- Plugin stays **zero-dependency**; skills frontmatter = `name`+`description` only.
- `claude plugin validate --strict` green in **both** modes; `marketplace.json` version-less.
- `plugin.json` version ↔ git tag in lockstep (the tag/release is the owner's step).
- The new guard is **pure stdlib**; no frozen eval artifacts touched; no `.claude/`.
